実行プラン

プロジェクト方針

本プロジェクトでは、後藤さんにプロジェクトリーダーとして全体を推進いただきます。

SEO戦略の設計、施策の優先順位判断、コンテンツ品質の担保、効果測定と軌道修正 —— プロジェクトの「頭脳」として、事業成長をドライブしていただくポジションです。

一方、サイトへの実装作業（記事やタイトルの書き換え、テーマの修正、リダイレクト設定、データ取得・集計など）はAIツール（Claude）が担当します。 テクニカルな処理・開発をAIに任せることで、後藤さんには戦略立案・品質管理・プロジェクト推進に集中していただける体制です。

役割分担

後藤さん
プロジェクトリーダー		 戦略・判断・品質管理を統括
・SEO戦略の設計と優先順位の決定
・競合分析・SERP調査に基づく施策方針の策定
・記事コンテンツの品質チェック・改善指示
・CTA / CV導線の設計
・被リンク獲得の企画・外部メディアとの関係構築
・効果測定と戦略の軌道修正
・社内チームへの進捗共有・報告
Claude（AI）
実装・処理担当		 後藤さんの方針に基づき、サイトへの実装作業を実行
・WordPress上での記事タイトル・本文・CTA等の書き換え
・テーマテンプレートの修正（HTML/CSS/JS）
・リダイレクト設定、canonical修正、noindex設定等のテクニカル処理
・Search Console / GA4 からのデータ取得・集計・レポート生成
・画像変換、内部リンク挿入などの一括処理
社内チーム
コンテンツオーナー		 ・監修者情報・写真の提供
・専門記事の執筆・最終レビュー
・ビジネス方針の最終承認

想定稼働

後藤さんには週10〜15時間程度の稼働をお願いできればと考えております。
主な活動は、戦略設計・コンテンツの品質管理・施策の推進管理・外部施策・効果測定とレポーティングです。
フェーズの進行やタスク量に応じて柔軟に調整いただければと思います。

Phase 0：現状把握と戦略策定（1週目）

まずサイトの全体像を把握し、今後の戦略の土台を作ります。

#タスク担当内容
0-1 サイトデータの一括調査 Claude 以下を並列で取得・整理：
・全144記事の振り分け表（タイトル・現カテゴリ・主要流入KW・URL変更有無）
・カニバリ記事ペア一覧
・2ページ目滞留KWリスト（順位10〜20）
・404エラー・インデックス未登録URL一覧
・内部リンク構造マップ
・Core Web Vitalsのボトルネック特定
0-2 競合サイトのSEO分析 後藤さん Movin / MyVision / コンサルキャリア / 外資就活 等の主要競合を調査し、Strategistsとの差分を整理：
・サイト構造・カテゴリ設計のパターン
・CTA配置・文言・デザイン
・著者情報・E-E-A-Tの見せ方
→ Strategistsに取り入れるべき点を整理
0-3 データレビューと戦略方針の策定 後藤さん Claudeの調査結果を確認し、今後の施策方針を決定：
・振り分け表のレビュー・修正（判断が微妙な記事の最終決定）
・カニバリペアの処理方針（統合 or 差別化）
・Phase 1以降の実行順序と優先度の確定
0-4 KW戦略マップの作成 後藤さん 調査データと競合分析を組み合わせ、3ヶ月間のKW攻略ロードマップを策定：
・即効性の高いKW（11-15位 → 1ページ目を狙う）
・新規記事で狙うべきKW（競合が弱い・CVに近い）
・優先度を下げるKW（競合が強すぎる・CVから遠い）
Phase 1：テクニカルSEO修正（2〜3週目）

サイトの技術的な問題を解消し、Googleに正しく評価される土台を整えます。

#タスク担当内容
1-1 テクニカル修正の実装 Claude 後藤さんの方針に基づき、以下を実装：
・404エラー58件の301リダイレクト設定
・canonicalタグの修正（#tocアンカー問題の解消）
・tag / page のnoindex設定
・画像WebP変換・LCP改善（loading="lazy"、fetchpriority設定）
・CSS/JSの遅延読み込み設定
1-2 実装結果の検証・品質確認 後藤さん Claudeの実装が正しく反映されているかを検証：
・PageSpeed Insightsでのスコア再計測
・主要URLのリダイレクト動作確認
・GSCでのインデックス状況の変化確認
→ 問題があればClaudeに修正を指示
1-3 記事タイトル改善案の作成 Claude 全144記事のタイトル改善案を一括生成
1-4 タイトル改善案の精査・確定 後藤さん Claudeが生成したタイトル案を1件ずつ精査：
・検索意図との整合性チェック
・CTRを高める表現の調整
・シリーズ番号「①②③」の代替表現を決定
→ 確定リストをClaudeに渡し、WordPressに一括反映
1-5 CTA・CV導線の設計 後藤さん 現CTAの課題を踏まえ、新しいCV導線を設計：
・記事カテゴリ別のCTAパターン（ケース面接→無料相談、企業研究→LP等）
・CTA文言の作成（3〜4パターン）
・配置ルールの策定（記事中+末尾のどこに何を置くか）
→ Claudeが全記事に一括実装
Phase 2：カテゴリ再編（4〜6週目）

サイト構造を整理し、Googleの評価分散（カニバリゼーション）を解消します。

#タスク担当内容
2-1 カテゴリ作成・記事移行の実行 Claude ・新カテゴリ「career」作成
・まず流入ゼロ記事（約11記事）でテスト移行
・その後、後藤さんの確定リストに基づき1日5記事ペースで段階移行
2-2 移行の進捗管理・判断 後藤さん 移行期間中、Search Consoleの順位変動を確認しながらプロジェクトを推進：
・順位下落が発生した場合の対応方針決定（ロールバック or 静観）
・インデックス再登録のタイミング判断
・移行ペースの調整（加速 or 一時停止）
2-3 カニバリ記事の処理方針決定 後藤さん カニバリペアごとに最適な対応を判断：
・統合する場合：残す記事の選定、取り込むコンテンツの取捨選択
・差別化する場合：各記事のターゲットKW再設定
→ Claudeがリダイレクト設定 or 記事の書き換えを実行
2-4 2ページ目滞留KWの記事強化 後藤さん KW戦略マップから優先記事を選定し、改善方針を策定：
・上位記事との差分分析（構成、情報量、E-E-A-T等）
・タイトル・H1・冒頭300字・見出し構成の改善指示
・内部リンクの追加先指定
→ Claudeが改善案を生成 → 後藤さんが最終チェック → Claudeが実装
2-5 旧カテゴリの削除 Claude 全記事の移行完了後、後藤さんの確認を経て実行
Phase 3：コンテンツ品質改善（5〜8週目）

記事単体の品質を向上させ、E-E-A-T（経験・専門性・権威性・信頼性）を強化します。

#タスク担当内容
3-1 E-E-A-T戦略の設計 後藤さん 著者情報・監修者情報の見せ方を設計：
・監修者ボックスの掲載情報（名前、肩書、写真、一言コメント）
・記事タイプ別の監修者割り当てルール
・著者プロフィールページの構成
→ Claudeがテーマテンプレート + JSON-LD構造化データを実装
3-2 著者ボックス・構造化データの実装 Claude テーマにカスタムフィールド追加、著者ボックスHTML/CSS、JSON-LD Personスキーマを実装
3-3 CTA一括実装 Claude Phase 1で後藤さんが設計したCTAパターンを全記事に反映
3-4 リライト対象の選定と品質レビュー 後藤さん データに基づいてリライトの優先順位を決定し、品質を管理：
・上位表示だがCTR低い記事のタイトル・メタディスクリプション改善
・内容が古くなった記事の更新方針策定
・競合上位記事との比較で不足セクションを特定
→ Claudeがリライト案を生成 → 後藤さんが品質チェック → Claudeが反映
3-5 YouTube埋め込みの改善 Claude テキストリンク → サムネイル+タイトル表示のカードUIに変更
3-6 新卒/中途セグメントの設計 後藤さん ターゲット別のコンテンツ・導線を整理：
・記事タイトルへの【新卒向け】【中途向け】表記ルール策定
・LP分離の要否判断
・カテゴリ内フィルタリングの設計
Phase 4：内部リンク・サイト構造強化（7〜10週目）

サイト内の記事同士のつながりを強化し、Googleの評価を重要ページに集中させます。

#タスク担当内容
4-1 トピッククラスターの設計 後藤さん カテゴリごとのピラーページと内部リンク構造を設計：
・ピラーページの構成（「ケース面接 完全ガイド」「フェルミ推定 完全ガイド」等）
・各ピラーからサブ記事への内部リンク設計図
・ピラーページのターゲットKW・見出し構成
4-2 関連記事のキュレーション 後藤さん 主要記事30本について「この記事の次に読むべき記事」を選定。
自動プラグインではなく、検索意図に沿った導線を編集判断で設計。
4-3 内部リンク・関連記事・パンくずの実装 Claude 後藤さんの設計に基づき一括実装：
・記事本文中へのアンカーテキストリンク挿入
・記事末尾の関連記事セクション追加
・パンくずリスト + JSON-LD BreadcrumbList追加
4-4 カテゴリまとめページの企画 後藤さん 4カテゴリ分のランディングページを設計：
・ページ構成（導入文、人気記事、目的別おすすめ等）
・ターゲットKW・メタ情報
・CTA配置
4-5 カテゴリまとめページの実装 Claude 後藤さんの構成に基づきテンプレートを実装。社内チームが導入文を最終調整。
Phase 5：新規コンテンツ・外部施策・継続改善（11週目〜）

新規コンテンツの拡充と外部評価の向上を進め、安定的な成長基盤を構築します。

#タスク担当内容
5-1 新規記事の企画・ディレクション 後藤さん 月4本の新規記事を企画・管理：
・競合ギャップ分析に基づくKW選定
・見出し構成案の作成（差別化ポイントの設計）
・社内チームへの執筆依頼と品質レビュー
5-2 記事リライトのディレクション 後藤さん 月2〜3本のリライトを管理：
・対象記事の選定（データに基づく優先順位）
・リライト方針の策定（追加/削除/構成変更）
・品質チェックと公開判断
・公開後の順位変動モニタリング
5-3 被リンク獲得施策 後藤さん サイトの外部評価を高める施策を推進：
・被リンク候補サイトのリストアップ（就活メディア、コンサル業界メディア等）
・ゲスト記事・インタビュー記事の企画
・PR記事・プレスリリースの企画
5-4 効果測定・戦略の軌道修正 後藤さん 週次で施策効果を振り返り、戦略を調整：
・KPIの進捗確認（順位、流入数、CV数）
・施策効果の分析（何が効いているか、改善余地はどこか）
・次週の優先タスク調整
・社内チームへの進捗報告
5-5 レポート生成・新カテゴリ追加 Claude ・GSC + GA4 APIで月次SEOレポートを自動作成
・新規記事が揃った段階で「コンサル業界」カテゴリを追加
